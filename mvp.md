# 習慣化アプリ MVP 設計まとめ

## ① MVP 機能優先度表

### 🔴 P0（必須 / MVP成立条件）
| 機能 | 優先度 | 補足 |
|---|---|---|
| 月次グリッド表示（トップ） | ★★★★★ | アプリの世界観そのもの |
| 習慣の作成 / 編集 / 削除 | ★★★★★ | 利用の前提 |
| 1タップチェック（成功 / 未実施） | ★★★★★ | 継続率に直結 |
| 今日の行の自動ハイライト | ★★★★☆ | 思考コスト削減 |
| ローカル永続化 | ★★★★☆ | オフライン前提 |
| 月切り替え | ★★★★☆ | 実運用で必須 |

---

### 🟡 P1（余力があれば / 後回し可）
| 機能 | 優先度 | 注意点 |
|---|---|---|
| Streak（連続日数） | ★★★☆☆ | 表示は控えめに |
| 月末サマリー（自動） | ★★★☆☆ | 手入力不要 |
| 習慣テンプレ | ★★☆☆☆ | 初期入力を重くしない |
| リマインド通知 | ★★☆☆☆ | OFFがデフォルト |

---

### 🔵 P2（初期に入れると失敗しやすい）
| 機能 | 理由 |
|---|---|
| 詳細グラフ・分析 | 利用頻度が低い |
| AI改善提案 | 不安・義務感を煽る |
| ランキング | 習慣が競争化する |
| コメント | 運用コスト増 |
| 複雑な数値入力 | 継続率低下 |

---

### ❌ 意図的に作らない機能
- 過去達成率の強調表示
- 未達成時の警告・通知
- バッジや過度な演出

> 方針：責めない・考えさせないUI

---

## ② マネタイズ設計（無料 / 有料境界）

### 基本思想
- 習慣そのものを人質にしない
- 記録・閲覧は常に無料
- 課金価値は「安心・拡張・共有」

---

### 🆓 無料プラン（永続）
| 項目 | 内容 |
|---|---|
| 習慣数 | 最大3 |
| 月次グリッド | ○ |
| チェック記録 | ○ |
| 月切り替え | ○ |
| ローカル保存 | ○ |
| 広告 | なし |

---

### 💰 有料プラン（Pro）
| 機能 | 内容 |
|---|---|
| 習慣数 | 無制限 |
| クラウド同期 | ○ |
| 端末変更時の復元 | ○ |
| 月次サマリー保存 | ○ |
| ゆる共有（閲覧のみ） | ○ |

---

### 👥 チーム機能（上位課金限定）
| 機能 | 条件 |
|---|---|
| グループ作成 | Pro以上 |
| 他人の実施可視化 | Pro以上 |
| 応援リアクション | Pro以上 |

---

| プラン | 目安価格 |
|---|---|

---

### ⚠️ よくある失敗例
- 無料で習慣数無制限
- 記録データ閲覧を有料化
- 共有・競争を前面に出す

---

### 推奨リリース戦略
1. 完全無料・ローカル版で公開
2. クラウド同期のみ有料化
3. 少人数共有を後出し実装
4. 3週間継続ユーザーにのみ課金導線を表示
